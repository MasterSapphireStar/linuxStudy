# Ubuntu 下 C/C++ 开发配置使用说明

## ⚠️ 重要说明 - Ubuntu 的限制

**坏消息**：Ubuntu 下 VS Code 的 C++ 调试器对外部终端支持不完善。
按 F5 使用任何 cppdbg 配置都会显示 GDB 后台信息：
```
100
[1] + Done    "/usr/bin/gdb" --interpreter=mi ...
```

**好消息**：有替代方案可以获得干净的输出！

---

## 🎯 推荐使用方法（避免 GDB 信息）

### ⭐ 方法 1：使用 run-external 任务（最接近 Windows 体验）

**操作步骤：**
1. 按 `Ctrl + Shift + P`
2. 输入 "Tasks: Run Task"
3. 选择 **"run-external"**

**效果：**
- ✅ 弹出独立的 GNOME Terminal 窗口
- ✅ 输出干净，无 GDB 信息
- ✅ 自动编译并运行
- ✅ 类似 Windows 的 cmd 窗口体验
- ✅ 程序结束后等待按键

---

### 📝 方法 2：使用 run 任务（集成终端）

**操作步骤：**
1. 按 `Ctrl + Shift + B`
2. 选择 **"run"**

**或者：**
1. 按 `Ctrl + Shift + P`
2. 输入 "Tasks: Run Task"
3. 选择 **"run"**

**效果：**
- ✅ 在 VS Code 集成终端运行
- ✅ 输出干净，无 GDB 信息
- ✅ 自动编译并运行
- ✅ 程序结束后等待按键

---

### 🐛 方法 3：调试模式（打断点用）

**操作步骤：**
1. 在代码中设置断点（点击行号左侧）
2. 按 `F5`
3. 选择 **"C/C++ 调试(GDB)"**

**效果：**
- ✅ 可以打断点、单步执行
- ✅ 查看变量值
- ⚠️ printf 输出在"调试控制台"面板

---

## 📊 使用场景对照表

| 需求                   | 推荐方法                  | 快捷操作                      | 是否弹窗 | 是否干净 |
| ---------------------- | ------------------------- | ----------------------------- | -------- | -------- |
| 🪟 弹出独立终端查看输出 | **run-external 任务** ⭐  | Ctrl+Shift+P → run-external   | ✅       | ✅       |
| 📝 集成终端查看输出     | **run 任务**              | Ctrl+Shift+B → run            | ❌       | ✅       |
| 🐛 调试（打断点）       | **F5 → C/C++ 调试(GDB)**  | F5 → 选调试配置               | ❌       | N/A      |
| ⚠️ F5 运行（不推荐）    | ~~F5 → C/C++ 运行~~       | ~~会显示 GDB 信息~~           | ✅       | ❌       |

**结论**：想要干净输出，用 **run-external** 或 **run** 任务，不要用 F5 运行配置！

## ⚙️ 配置文件说明

### tasks.json（任务配置）⭐

- **build**: 编译当前文件（自动识别 .c 或 .cpp）
- **run**: 在集成终端运行，干净输出 ✅
- **run-external**: 弹出独立终端运行，干净输出 ✅ 最接近 Windows
- **compile-c**: 强制使用 gcc 编译
- **compile-cpp**: 强制使用 g++ 编译

### launch.json（调试配置）

- **C/C++ 运行(外部终端)**: ⚠️ 会显示 GDB 信息，不推荐用于查看输出
- **C/C++ 调试(GDB)**: 用于调试，可打断点 ✅

## 💡 常见问题

### 1. 为什么按 F5 会显示 GDB 信息？

这是 VS Code 在 Linux 下的已知限制。所有 `cppdbg` 类型的配置都必须通过 GDB 运行，
即使设置了 `externalConsole: true` 也会在终端显示 GDB 的后台进程信息。

**解决方案**：使用 task（run-external 或 run）代替 F5 调试配置。

### 2. 如何获得最接近 Windows 的体验？

使用 **run-external** 任务：
- 按 `Ctrl + Shift + P`
- 输入 "Tasks: Run Task"
- 选择 "run-external"

这会弹出独立的终端窗口，就像 Windows 的 cmd 一样。

### 3. printf 后面没有换行怎么办？

在 printf 中加上 `\n`：
```c
printf("%d\n", num);  // 推荐
```

或使用：
```c
printf("%d", num);
fflush(stdout);
```

### 4. 快捷键速查

- `Ctrl + Shift + B`: 构建/运行任务菜单
- `Ctrl + Shift + P`: 命令面板（运行任务）
- `F5`: 调试（只用于打断点调试）
