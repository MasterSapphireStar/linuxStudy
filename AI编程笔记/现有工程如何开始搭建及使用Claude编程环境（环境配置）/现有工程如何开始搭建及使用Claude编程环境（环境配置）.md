# 现有工程如何开始搭建及使用Claude编程环境（环境配置）

## 准备工作

在开始配置之前，请确保以下条件已满足：

| 条件 | 说明 |
|:-----|:-----|
| 系统环境 | Ubuntu 22.04 或更高版本 |
| API密钥 | Claude 环境配置所需的 API Key |
| 编码转换 | 工程源码统一转换为 UTF-8 编码（建议全工程同步转换） |

> [!warning] 特别注意：中文字符串的使用场景
> 工程中涉及中文使用时需格外注意以下两类场景：
> - 用于 TTS 语音播放的中文字符串
> - 用于 LCD 屏幕显示的中文字符串

---

## 一、源码编码转换

在将旧工程接入 Claude 编程环境前，需确保源码编码统一为 **UTF-8 无BOM格式**。

### 1.1 转换工具

提供专用的 Python 转换脚本 `convert_to_utf8.py`，可自动识别并转换多种常见编码。

**支持的源文件编码：**

| 编码类型 | 具体编码 |
|:---------|:---------|
| 简体中文 | `gb2312`、`gbk`、`gb18030` |
| 繁体中文 | `big5` |
| 西欧编码 | `latin-1`、`cp1252` |

**支持的文件扩展名：**

| 语言 | 扩展名 |
|:-----|:-------|
| C/C++ | `.c`、`.h`、`.cpp`、`.hpp`、`.cc`、`.cxx`、`.c++`、`.hh`、`.hxx` |
| Python | `.py`、`.pyw` |
| Java | `.java`、`.jsp`、`.jspx` |
| JS/TS | `.js`、`.jsx`、`.ts`、`.tsx`、`.mjs`、`.cjs` |
| Go | `.go` |
| Rust | `.rs` |
| Ruby | `.rb`、`.erb` |
| PHP | `.php`、`.phtml` 等 |
| Shell | `.sh`、`.bash`、`.zsh`、`.fish` |
| 其他 | `.cs`、`.swift`、`.kt`、`.sql`、`.lua`、`.pl` 等 |

### 1.2 常见问题：编码混合导致的中文异常

> [!danger] 编码混合风险
> 当代码中存在部分 GB2312、部分 UTF-8 混合编码时，中文传递会出现异常。

**高风险场景：**
- 中文字符串传入 TTS 语音播放功能
- 中文字符串传输给 LCD 屏幕显示

**检索代码中的中文字符串（需人工排查）：**
![[Pasted image 20260313091756.png]]

### 1.3 解决方案

| 方案 | 说明 | 适用场景 |
|:-----|:-----|:---------|
| 方案一 | 源码统一修改为 UTF-8 格式 | 推荐首选，从根本上解决 |
| 方案二 | 使用统一转换函数 | 需保持下游兼容时 |
| 方案三 | 硬编码为 GB2312 | 特定硬件要求时 |

## 二、项目结构配置

根据项目实际情况，可选择以下两种组织方式：

### 方式一：配置工程内嵌源码目录（推荐多项目复用）

在 Claude 配置工程中建立源码子目录，将原有工程源码纳入管理。

| 特点 | 说明 |
|:-----|:-----|
| 优势 | 通用的 Skill 可在多个工程间复用 |
| 注意 | 需使用独立的 Git 分支进行版本管理 |

**结构示意：**
![[Pasted image 20260308182700.png]]

### 方式二：配置工程与工程根目录合并（推荐单项目专注）

将 Claude 配置工程直接置于原有工程的根目录下，与源码共存。

| 特点 | 说明 |
|:-----|:-----|
| 优势 | 结构简单，配置与源码紧密结合 |
| 适用 | 专注于单个工程的深度开发 |

## 四、Skill 使用指南

Skill 是 Claude 的扩展能力，可通过以下两种方式调用：

### 4.1 命令式调用（精准快速）

直接使用 `/skill名称` 格式调用，适用于已知具体 Skill 名称的场景。

![[Pasted image 20260308184311.png]]

### 4.2 自然语言描述（灵活便捷）

通过描述需求让 Claude 自动匹配 Skill。

**示例：** "使用 XXX Skill 来实现 MMMM 功能"

![[Pasted image 20260308184455.png]]

### 4.3 自定义 Skill 的创建

> [!tip] 创建工具
> 使用专门的 `skill-creator` Skill 来创建新 Skill，可获得更好的生成效果。

**创建方法：** 只需描述所需创建 Skill 的用途即可。

**示例：** "创建一个用于自动生成代码注释的 Skill"

**适用场景：**
- 需要重复执行的固定流程
- 具有明确输入输出的标准化任务
- 团队协作中需要统一规范的操作

## 五、最佳实践建议

对于 Claude CLI，你可以把它当成一个团队成员。与人沟通需要把话说清楚，与 AI 沟通也是一样——需求描述越详细，AI 就越能按你的想法去实现。

### 关键原则

| 原则 | 说明 |
|:-----|:-----|
| **先做 Plan** | 让 AI 先告诉你它计划怎么做，确认计划很重要 |
| **避免模糊需求** | 不要指望 AI 能完全理解并执行一句模棱两可的要求 |
| **详细描述需求** | 越详细越好，包含背景、约束、预期输出等 |

> [!quote] 找找使用 AI 编码的感觉
> 与 Claude 协作是一个逐步磨合的过程。随着使用经验的积累，你会越来越擅长与 AI 高效沟通，发挥 AI 编程的最大价值。
