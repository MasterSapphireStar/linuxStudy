# 第一部分：环境配置

> [!info] 现有工程搭建 Claude 编程环境

### 工程编码转码
- 旧工程源码大多为 GB2312 编码，Claude 等 AI 工具兼容性不好，需要转 UTF-8
- 转码脚本介绍：支持的编码、支持的文件后缀

### 工程结构设计方式
- **方案一**：配置工程内嵌源码目录（通用 skill 可方便复用）
- **方案二**：配置工程与工程根目录合并（跟随工程走）

### 工程初始化
- `/init` 操作演示
- 第一次可描述要求使用中文描述，自动扫描工程结构，生成配置
- `CLAUDE.md` 跟工程走

### Skill 基础使用
- **命令方式**：`/skill名称`
- **自然描述**：使用 XXX Skill 来实现功能

# 第四部分：总结

> [!summary] 核心要点

### 操作流程
```
输入需求 → 方案选择 → 计划确认 → 执行生成 → 结果确认 → Git 提交归档
```

### 关键经验
- ✅ 输入质量决定输出质量
- ✅ 分步确认很重要
- ✅ 及时版本管理
- ✅ Plan 模式下当设计发生偏差，可以重新总结补充需求，clear 后重新计划，减少幻觉

### 落地建议
- 🚀 从小功能开始尝试
- 🚀 建立项目专属 Skill 库
- 🚀 持续优化积累经验

> [!note] 提示
> 分享的是一个思路、用法，有一些通用 skill 或许可以复用，但实际上还是需要根据自己的项目进行调整和创造。
