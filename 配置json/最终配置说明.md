# Ubuntu 下 C/C++ 配置 - 最终版本

## 🎯 配置说明

现在的配置已经**尽可能**接近 Windows 版本：

### 配置 1: "C/C++ 运行环境"
- 按 F5 → 选择第一个配置
- 会显示编译过程（build 任务）
- 弹出外部终端显示程序输出
- ⚠️ VS Code 终端会显示 GDB 后台信息（无法避免）

### 配置 2: "C/C++ 调试(GDB)"  
- 按 F5 → 选择第二个配置
- 用于打断点调试

---

## ⚠️ 关于 GDB 后台信息

你看到的这些信息：
```
100
[1] + Done    "/usr/bin/gdb" --interpreter=mi ...
```

**原因**：
- Linux 下，VS Code 必须通过 GDB 运行程序
- GDB 作为后台进程运行，bash 会报告它的完成状态
- 这是 Linux 平台的限制，Windows 的 cppvsdbg 不会有这个问题

**解决方案**：

### 方案 1：隐藏 VS Code 的终端面板 ⭐ 推荐
按 F5 运行后，**关闭或最小化 VS Code 底部的终端面板**，只看弹出的外部终端窗口。

### 方案 2：使用 run-external 任务（完全干净）
不用 F5，改用：
- 按 `Ctrl + Shift + P`
- 输入 "Tasks: Run Task"  
- 选择 "run-external"

这个方法**完全没有 GDB 信息**，输出最干净！

### 方案 3：使用 run 任务（集成终端）
- 按 `Ctrl + Shift + B`
- 选择 "run"
- 在 VS Code 集成终端运行，输出干净

## 🎓 结论

**如果你习惯按 F5：**
1. 按 F5 选择 "C/C++ 运行环境"
2. 程序会在外部终端显示输出
3. 忽略或关闭 VS Code 底部终端的 GDB 信息

**如果你想要最干净的输出：**
1. 按 `Ctrl + Shift + P`
2. 输入 "Tasks: Run Task"
3. 选择 "run-external"
4. 完全没有 GDB 信息！

---

## 💡 为什么 Ubuntu 和 Windows 不同？

| 平台    | 调试器类型 | 运行方式             | GDB 信息 |
| ------- | ---------- | -------------------- | -------- |
| Windows | cppvsdbg   | 直接运行或 MinGW GDB | 无       |
| Linux   | cppdbg     | 必须通过 GDB         | 有       |

这是平台设计差异，不是配置问题！
