# ⚠️ 重要！Ubuntu 下正确使用 VS Code 运行 C/C++ 程序

## 🔴 错误方法（你一直在用的）

❌ 按 `F5` → 会显示 GDB 信息：
```
100
[1] + Done    "/usr/bin/gdb" --interpreter=mi ...
```

**原因**：F5 启动的是调试器，在 Linux 下必定通过 GDB，无法避免这些信息。

---

## ✅ 正确方法（干净输出）

### 方法 1：使用 Ctrl+Shift+B（最简单）

1. 按 `Ctrl + Shift + B`
2. 选择 **"run"**
3. 看到干净输出：
   ```
   100
   按任意键继续...
   ```

### 方法 2：使用命令面板

1. 按 `Ctrl + Shift + P`
2. 输入 "Tasks: Run Task"
3. 选择 **"run"**

---

## 🎯 记住这个

在 Ubuntu 上：
- **F5 = 调试器 = 有 GDB 信息 = 只用于打断点调试**
- **Ctrl+Shift+B → run = 直接运行 = 干净输出 = 用于查看 printf**

---

## 📋 对照表

| 需求               | Windows 习惯 | Ubuntu 正确方法              |
| ------------------ | ------------ | ---------------------------- |
| 查看 printf 输出   | F5           | Ctrl+Shift+B → run           |
| 调试（打断点）     | F5 调试      | F5 → C/C++ 调试(GDB)         |

---

## 💡 总结

**不要再按 F5 查看输出了！**

在 Ubuntu 下：
- 想看输出 → **Ctrl+Shift+B → run**
- 需要调试 → **F5 → C/C++ 调试(GDB)**

这是 Linux 和 Windows 的区别，没办法改变。
