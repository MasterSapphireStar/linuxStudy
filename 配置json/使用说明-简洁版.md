# Ubuntu 下 C/C++ 开发 - 快速使用指南

## 🎯 两种使用方式

### ⭐ 方式 1：运行查看输出（推荐）

**操作：** 按 `Ctrl + Shift + B` → 选择 **"run"**

**效果：**
```
100
按任意键继续...
```
- ✅ 输出干净，无 GDB 信息
- ✅ 在 VS Code 集成终端显示
- ✅ 自动编译并运行
- ✅ 程序结束后等待按键

---

### 🐛 方式 2：调试（打断点）

**操作：** 按 `F5` → 选择 **"C/C++ 调试(GDB)"**

**用途：**
- ✅ 打断点、单步执行
- ✅ 查看变量值
- ⚠️ printf 输出在"调试控制台"面板

---

## 📋 快速记忆

| 🐛 调试打断点 | `F5` → C/C++ 调试(GDB) |

## �� 为什么不用 F5 查看输出？

F5 的所有配置都通过 GDB 调试器运行，会显示：
```
100
[1] + Done    "/usr/bin/gdb" --interpreter=mi ...
```

**解决方案**：用 run 任务代替 F5，输出干净！

---

## 🔧 可用的任务

按 `Ctrl + Shift + B` 可以看到：

- **run** ⭐ - 编译并运行，集成终端显示输出
- **run-external** - 编译并运行，弹出独立终端窗口
- **build** - 只编译，不运行
- **compile-c** - 强制使用 gcc 编译
- **compile-cpp** - 强制使用 g++ 编译
